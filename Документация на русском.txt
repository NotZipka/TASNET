внутри [] значение, необходимо писать только значение, без скобок.
вместо {} текст команды.


bot [количество ботов] - указывает количество ботов на сервер (добавляет если число больше чем ботов на сервере и наоборот).
bot+ - добавляет одного бота.
bot- - удаляет одного бота.

more_info [выкл/вкл(0/1)] - пишет в чат действия совершенные из консоли (например бот(0)_удар 1 или бот(12)_прыжок 0). можно управлять через autoexec_server.
more_info [выкл/вкл(0/1)] - пишет в чат конкретный кадр.
sv_get_auto_access [выкл/вкл(0/1)] - при заходе на сервер автоматически даёт доступ к правам администратора человеку под нулевым айди. можно управлять через autoexec_server.
bot_on [айди бота] [не бот, бот/выкл, бот/вкл(-1/0/1)] - режим активной записи действий(вкл), режим без записи действий(выкл).
bot_on_multiple [айди первого бота] [айди последнего бота] [не бот, бот/выкл, бот/вкл(-1/0/1)] - работает как bot_on, но меняет состояние сразу нескольким ботам (А и Б тоже меняет).
bot_spec [айди бота] - помещает бота в наблюдатели (/spec). не использовать при создании, так как работает некорректно. в будущих версиях будет работающий вариант, пока что нет.
bot_hookcoll [выкл/вкл(0/1)] - показывает колизию хука ботов.
bot_state_{deep/freeze/solo} [айди бота] [выкл/вкл(0/1)] - изменяет напрямую конкретное состояние бота !ОСТОРОЖНО - МОЖЕТ ПРИВОДИТЬ К НЕВОЗМОЖНЫМ СИТУАЦИЯМ! Использовать только если вы уверены на 100% что произошла ошибка в расчетах(не в ваших, например при отмотке назад).
bot_team_on  [номер тимы] - устанавливает стандартную команду для активных ботов (bot_on 1), чтобы не взаимодействовать с неактивными ботами. можно управлять через autoexec_server.
bot_team_off [номер тимы] - устанавливает стандартную команду для неактивных ботов (bot_on 0), чтобы не взаимодействовать с активными ботами. можно управлять через autoexec_server.
bot_tp [айди бота] - телепортирует бота в центр твоего экрана.

bot_i_dir	 [айди бота] [влево/стоп/вправо(-1/0/1)] - направление движения бота в следующем кадре.
bot_i_jump	 [айди бота] [стоп/прыжок(0/1)] - прыжок бота.
bot_i_hook	 [айди бота] [стоп/хук(0/1)] - хук бота.
bot_i_fire	 [айди бота] [стоп/удар(0/1)] - удар/выстрел бота.
bot_i_weapon	 [айди бота] [молоток/пистолет/шотган/рокет/лазер(1/2/3/4/5)] - выбор оружия бота.
bot_i_angle	 [айди бота] [угол(0-360.00)] - ставит угол прицела бота, в градусах (до сотых).
bot_i_angle_plus [айди бота] [угол(0-360.00)] - добавляет к углу значение, а не устанавливает.
bot_i_aim_target [айди бота] [айди цели] - бот целится в другого бота (цель)
bot_i_aim_spec	 [айди бота] - бот целиться в твой центр экрана (то есть когда ты в спеке, он будет целиться туда где твоя камера).
bot_i_reset	 [айди бота] - сбрасывает зажатые действия бота.
bot_i		 [айди бота] [влево/стоит/вправо(-1/0/1)] [стоит/прыжок(0/1)] [нету/хук(0/1)] [нету/удар(0/1)] [оружие(1-5)] [угол(0-360.00)]- все одновременно.
bot_i2		 [айди бота] [влево/стоит/вправо(-1/0/1)] [стоит/прыжок(0/1)] [нету/хук(0/1)] [нету/удар(0/1)] [оружие(1-5)] [нет/целиться в центр экрана(0/1)] - все одновременно, но другой прицел.

b_{input/dir/jump/hook/fire/weapon/angle/angle_plus/aim_target/aim_spec/reset} [состояние] - те же команды с теми же значениями только без айди, то есть будет присваивать значения для всех включенных в данный момент ботов.

tas_step  [0-10000] - запускает время на определенное количество тиков (в 1 секунду ровно 50 тиков). Удаляет все следующие записанные действия для включенного бота, для выключенного работает так же как tas_redo.
tas_undo  [0-10000] - отматывает назад время на определенное количество кадров (если таковых нет - загрузит самый первый). Для всех ботов в ТАС.
tas_redo  [0-10000] - возвращает вперед время на определенное количество кадров (если таковых нет - загрузит самый последний). Для всех ботов в ТАС.
tas_plus  [0-10000] - добавляет кадр при этом все последующие кадры сдвинутся вперед на то же количество. Для выключенного бота работает как tas_undo.
tas_minus [0-10000] - удаляет кадр при этом все последующие кадры сдвинутся назад на то же количество. Для выключенного бота работает как tas_redo.

tas_save     [0-100000]  - сохраняет весь ТАС. Введенное число указывает на слот сохранения (не кадр...), без символов.
tas_autosave [0 - 10000] - автоматически сохраняет ТАС если ты достигаешь кадра который делится на введённое число (сохраняет только при добаавлении). 0 выключает. стандартно делает сохранение каждые 200 кадров.

tas_load_old [0-10] - позволяет загружать сохранения из предыдущих версий. 0 - текущая версия. 1 - для версий 2.х.
tas_load  [0-100000]  - загружает весь ТАС. Введенное число указывает на слот сохранения (не кадр), без символов.
tas_frame [0-1000000] - загружает конкретный кадр.

tas_play  [1-1000000] - запускает ТАС с определенного кадра.
tas_speed [1-100]     - стандартная скорость 10, если меньше - замедляет, если больше - ускоряет.


Кастомизация и тп:
tas_mode_crazy       [0/1] — во время воспроизведения бот сходит с ума.
tas_mode_relax       [0/1] — заставляет бота медленно перемещать перекрестие во время воспроизведения.
tas_mode_auto_closer [кадры] — перемещает прицел бота в нужную точку за определенное количество кадров.

bot_name        [айди бота] [имя] - позволяет менять имя любого бота.
bot_clan        [айди бота] [клан] - позволяет менять клан любого бота.
bot_skin        [айди бота] [цвет(0/1)] [цвет тела] [цвет ног] - позволяет менять скин любого бота.
bot_emote       [айди бота] [эмоция(0-15)] - вызывает эмоцию бота.
bot_emote_delay [айди бота] [тик(=<0)] - количество тиков через которое бот спамит последнюю эмоцию. 0 - выкл.

Важное уточнение:
ВСЕ команды сделанные по типу tas_input являются не более чем копией команд bot_i, с той лишь разницей что значения которые они принимают охватывают одновременно несколько действий, те кому будет удобно ими 
пользоваться в инструкции не нуждаются, я уверен.
